# Product

## Register

product

## Users

OpenGUI is for non-specialist users and developers who want to use AI coding agents with less terminal juggling and more visible control. They are often managing real work across projects, sessions, models, and tools, and need the interface to stay understandable during long agent runs.

## Product Purpose

OpenGUI provides a desktop and web command center for coding-agent harnesses. It lets users connect projects, choose agents and models, send prompts, queue follow-up work, review streaming responses, and manage agent workflows from one UI. Success means users feel oriented, can act confidently, and can keep multi-step AI work moving without losing context.

## Brand Personality

Calm, practical, technical. The product should feel capable and trustworthy without becoming flashy or unfamiliar. Improvements should preserve the existing design language and focus on clarity, contrast, and color discipline.

## Anti-references

Avoid generic SaaS dashboard gloss, washed-out gray-on-gray interfaces, decorative gradients, over-rounded AI chat styling, and changes that make the app feel like a different product. Avoid high-chroma color everywhere; color should clarify state, hierarchy, and action.

## Design Principles

- Preserve the current product shape while improving legibility.
- Use contrast to reduce cognitive load during long sessions.
- Make primary actions and system states visible without visual noise.
- Keep controls dense enough for power workflows, but readable for non-specialist users.
- Let color communicate priority, status, and focus rather than decoration.

## Accessibility & Inclusion

Target WCAG AA contrast for text, placeholders, controls, borders, and focus states. Support reduced motion and avoid color-only state communication.
